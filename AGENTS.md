## Архитектура и важные ограничения

- Проект - Node.js бот на ESM. Используй `import`/`export`, не `require()`.
- Production рассчитан на Node.js 22, минимально допустимо Node.js >= 20.
- Основной вход: `src/index.js`.
- Telegram/VK логика лежит в `src/bots/`.
- Генерация песни идет через `src/flow/generate.js`.
- Telegram должен использовать очередь `src/queue.js`: одна SUNO-генерация за раз.
- Все настройки читаются централизованно через `src/config.js`; не добавляй прямое чтение `process.env` в другие модули.
- Сессии и платежи сейчас in-memory в `src/store.js`; после рестарта они теряются.
- Пользовательские ошибки не должны раскрывать backend: пиши "студия", а не "SUNO".

## SUNO, cookie и passkey

- Docker в production не использовать: `$` в cookies ломает env-файлы. Сервисы работают нативно через systemd.
- Cookie/passkey refresh должен быть on-demand, только при ошибках. Не включай обратно proactive timers `cookie-refresh.timer` и `passkey-refresh.timer` без явного решения.
- Cookie SUNO берутся через CDP из RDP Chromium на порту `9223`.
- Для cookie записывать только essential cookies, а не весь набор браузера, иначе возможна ошибка `Request Header Fields Too Large`.
- Не логируй cookies, токены, ключи, личные данные пользователя и полный пользовательский текст без необходимости.
- Не запускай Chromium для SUNO с `--user-data-dir=/tmp/...`: это создает пустой профиль без нужной сессии.

## Стиль кода

- JavaScript файлы в `src/` пишутся обычным `.js`, без TypeScript.
- Имена функций и переменных - `camelCase`.
- Константы уровня модуля - `SCREAMING_SNAKE_CASE`, только если это действительно постоянные значения.
- Отступ - 2 пробела.
- Строки - одинарные кавычки.
- Точки с запятой ставятся.
- Экспорты - именованные, без default export.
- Внутренние импорты должны явно заканчиваться на `.js`.
- Комментарии, логи и пользовательские тексты можно писать по-русски; имена переменных и функций оставляй на английском.

## Проверки

- После JS-изменений запускай:
  - `npm run check`
  - `npm test`, если менялась AI-логика, pipeline, metrics, critic или rewriter
- Можно запускать точечно:
  - `npm run test:metrics`
  - `npm run test:critic`
  - `npm run test:pipeline`
- Если менялся `services/rhyme-sidecar`, проверяй:
  - `cd services/rhyme-sidecar`
  - `./venv/bin/pytest tests/ -v` на Debian-сервере
  - или `python -m pytest tests -v`, если работаешь локально и окружение подготовлено
- Скрипты `test_*.mjs`, `get_*.mjs`, `intercept_*.mjs`, `scan_p1.mjs` работают с живой инфраструктурой и CDP. Не запускай их без понимания риска и без явного разрешения, если есть шанс потратить SUNO credits или дернуть production.
- Для docs-only правок достаточно проверить, что файл создан/прочитан корректно.

## Работа через WSL 20 и сервер

- Ты можешь работать через WSL 20 / Ubuntu-20.04.
- SSH на мини ПК:
  ```bash
  wsl -d Ubuntu-20.04 -- ssh alexander@192.168.0.128 'команда'
  ```
- Основной сервер: `192.168.0.128`, Debian 13.
- Важные systemd сервисы:
  - `podari-bot`
  - `suno-api`
  - `passkey-server`
  - `cloudflared`
  - `rhyme-sidecar`
  - `chromium-watchdog.timer`
- Не деплой и не рестартуй production-сервисы без явного разрешения пользователя.
- При разрешенном деплое типовой путь:
  ```bash
  git push origin main
  wsl -d Ubuntu-20.04 -- ssh alexander@192.168.0.128 'cd ~/projects/moyapesnya && git pull && npm install && sudo systemctl restart podari-bot'
  ```
- Логи смотреть так:
  ```bash
  wsl -d Ubuntu-20.04 -- ssh alexander@192.168.0.128 'sudo journalctl -u podari-bot --no-pager -n 80'
  ```

## Git и безопасность

- Перед правками смотри `git status --short`.
- Не откатывай чужие изменения и не удаляй незнакомые файлы.
- Не коммить `.env`, реальные токены, cookies, passkey, приватные ключи и пользовательские персональные данные.
- `.env.example` можно править как шаблон, `.env` трогай только если пользователь прямо попросил.
- Если работаешь с оплатой Robokassa, помни: подписи и пароли нельзя печатать в логах.

## Продуктовые правила

- Главная ценность продукта: человек должен услышать песню и узнать себя.
- AI lyrics pipeline обязан сохранять формат `{ lyrics, tags, title }`, потому что от него зависит SUNO.
- Качество текста важнее лишней скорости, но общий pipeline должен оставаться в разумных рамках ожидания пользователя, примерно до 2-3 минут.
- Не ухудшай пользовательский флоу Telegram без явной задачи: 5 вопросов, подтверждение, генерация текста, возможность правок, затем генерация песни.
- Robokassa/paywall сейчас может быть выключен через `PAYWALL_ENABLED=false`; перед изменениями оплаты сверяй актуальный статус в `.env.example`, `docs/progress.md` и коде.
