# AI Poet Pipeline — Подари Песню!

## What This Is

Улучшение системы генерации текстов песен для Telegram-бота «Подари Песню!». Сейчас один LLM пишет текст за один шаг — результат технически правильный, но мёртвый: банальные рифмы, нет истории, нет эмоций. Цель — построить multi-step pipeline (generate → critique → rewrite) и найти лучшую модель для русскоязычной поэзии через OpenRouter.

## Core Value

Когда человек слышит песню — он узнаёт себя и говорит «это точно про меня». Смеётся или плачет. Хочет отправить другу.

## Requirements

### Validated

- ✓ Single-step генерация (v11 промпт + Claude Sonnet 4.5) — работает, но качество низкое
- ✓ JSON output format: `{lyrics, tags, title}` — используется SUNO
- ✓ OpenRouter интеграция — src/ai/client.js
- ✓ Структура песни: [Куплет1][Припев][Куплет2][Бридж][Припев][Финал]
- ✓ Параметры: повод, жанр, настроение, голос, пожелания

### Active

- [ ] Исследование pipeline-архитектур: generate→critique→rewrite vs generate×N→judge
- [ ] Сравнение моделей на русской поэзии: o3, Gemini 2.5 Pro, Claude Opus, Mistral Large
- [ ] Реализация multi-step pipeline в src/ai/client.js
- [ ] Метрики качества: уникальность рифм, слоги, специфичность истории
- [ ] Набор тестовых кейсов для A/B сравнения (старый vs новый pipeline)

### Out of Scope

- Изменение SUNO интеграции — не связано с качеством текстов
- Изменение диалогового флоу бота — отдельная задача
- User-facing UI изменения — только backend pipeline
- Автоматическая оценка через реальных пользователей на данном этапе

## Context

**Текущий стек**: Node.js 22 ESM, OpenRouter API, модель `google/gemini-2.5-pro` с reasoning:high.

**Проблема с текущим промптом (v11)**: Несмотря на детальные инструкции и примеры хороших/плохих строф, модель всё равно выдаёт банальные рифмы (`любовь/кровь`), скучные истории и текст без живости. Промпт уже v11 — дальнейший тюнинг одного промпта даёт убывающую отдачу.

**Гипотеза**: Multi-step pipeline с отдельным AI-критиком который разносит текст по конкретным критериям (специфичность, рифмы, эмоция) и переписывает — даст качественный скачок. Критик видит конкретный плохой текст и объясняет ЧТО и ПОЧЕМУ не так, а не абстрактные правила.

**Бюджет времени**: Сейчас ~30-60 сек на генерацию. До 2-3 минут общего времени — приемлемо для пользователя.

**Доступные модели через OpenRouter**: Claude Opus 4.6, o3, Gemini 2.5 Pro/Ultra, Mistral Large, Qwen, DeepSeek и другие.

## Constraints

- **Latency**: Максимум 2-3 мин всего pipeline — пользователи ждут уже сейчас, дольше нельзя
- **Stack**: Node.js 22 ESM, src/ai/client.js — изменения только там
- **API**: OpenRouter (не Anthropic напрямую) — все модели через один endpoint
- **Cost**: Reasoning tokens дорогие — нужно балансировать качество vs стоимость
- **Output format**: JSON `{lyrics, tags, title}` должен сохраниться — SUNO зависит от него

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Multi-step pipeline вместо промпт-тюнинга | v11 — потолок одношагового подхода; критик видит конкретный текст, а не абстрактные правила | — Pending |
| OpenRouter для всех моделей | Единый endpoint, легко переключаться между моделями | ✓ Good |
| Качество важнее скорости | Пользователь покупает песню — плохой текст = возврат. До 3 мин ок | — Pending |
